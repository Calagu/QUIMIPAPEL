La base de datos del proyecto ha sido diseñada para gestionar el flujo completo de pedidos de una empresa de suministros de oficina.

El sistema permite administrar clientes, productos, usuarios y pedidos de forma centralizada.

Los clientes pueden realizar múltiples pedidos, y cada pedido puede incluir varios productos mediante líneas de detalle.  
Además, los pedidos pasan por diferentes estados durante su ciclo de vida, quedando registrado cada cambio en un historial de estados.

El sistema también incorpora gestión de usuarios y roles para controlar los permisos y funciones dentro de la aplicación, diferenciando entre administradores, comerciales, oficina y repartidores.

Por otro lado, se incluye un módulo de reparto que permite asignar pedidos a repartidores y controlar el estado de las entregas.

La base de datos también implementa automatizaciones mediante triggers y procedimientos almacenados para gestionar:
- control automático de stock,
- cálculo de importes,
- validaciones,
- y seguimiento de pedidos.
